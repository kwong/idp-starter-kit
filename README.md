# IDP Starter Kit

A production-grade Internal Developer Platform (IDP) starter kit designed to be spun up locally on Kubernetes before being promoted to EKS/GKE. Built entirely on a strict GitOps model via ArgoCD.

## Core Architecture

This starter kit provisions the following platform capabilities:
* **Continuous Deployment**: ArgoCD (App of Apps pattern)
* **Control Plane / Developer APIs**: Crossplane (`XApp` Compositions)
* **Secrets Management**: HashiCorp Vault + External Secrets Operator (ESO)
* **Identity & Access Management**: Keycloak (OIDC SSO with RBAC mapped groups)
* **Policy as Code**: Kyverno
* **Observability**: Prometheus, Grafana, Loki (Logs), Tempo (Traces), OpenTelemetry Collector

## Repository Layout

* `AGENTS.md`: Full architectural design document.
* `bootstrap/`: ArgoCD Applications ("App of Apps") that point to environments.
* `environments/`: Kustomize overlays that aggregate platform components for a specific environment.
* `components/`: "The Lego Bricks". Reusable base definitions for platform tools.
* `hack/`: Local development tools, scripts, and demo payloads strictly for `kind` testing.
* `./scaffold-env.sh`: Generates new environments based on `platform-values.yaml`.
* `./seed-vault-secrets.sh`: Seeds OIDC secrets into Vault KV for any environment.

## Quick Start (Local Run)

To deploy the entire platform locally, you must have Docker (or Colima), `kind`, `kubectl`, and `helm` installed.

```bash
# 1. Spin up the Kind cluster and run the GitOps bootstrap script
make run

# 2. Wait for ArgoCD to synchronize all platform apps
kubectl get applications -A -w

# 3. Access local UI components
make proxy # Proxies ArgoCD to https://localhost:8080 (admin / <auto-generated-password>)

# Grafana: http://grafana.local
# Vault: http://vault.local
# Keycloak: http://keycloak.local
```

### Testing the Developer Experience

To test that Crossplane is working successfully, simulate a developer requesting a new application:

```bash
kubectl apply -f hack/demo-xapp.yaml -n default
```

You can then run `kubectl get deploy,svc,ingress` to see the generated underlying primitives.
